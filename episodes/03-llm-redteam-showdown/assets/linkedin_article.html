I Attacked My Own AI Agent — Which Red-Team Tool Broke It First?

The scary thing about an AI agent isn’t that it can be jailbroken into saying something rude — it’s that it holds tools. It can fetch a URL, read a file, call an API. So the question that actually matters for anyone shipping one is: if an attacker talks my agent into misusing its tools, would the red-team framework I bought catch it before an auditor does? I built one deliberately weak agent, gave it a single benign canary tool, and pointed three of the best-known LLM red-team frameworks at it — Garak, PyRIT, and Promptfoo — on the exact same target, to see which one broke it first.

Everything runs locally in Docker. The target agent sits on an internal network with no egress; the only thing its tool can reach is an in-network canary sink. When the agent gets talked into firing that tool, the sink logs a hit — that hit is my ground truth that an attack actually landed, not a grader’s opinion. The whole thing is reproducible at a pinned commit.

Three tools that do genuinely different jobs

It’s tempting to treat these as three brands of the same product. They’re not. Garak is the scanner — think “Nessus for LLMs”: one command, a big battery of single-turn probes, run it like a vulnerability scan. PyRIT (Microsoft) is a framework for building multi-turn attacks — it points a second, “adversarial” LLM at your target and lets it escalate turn by turn. Promptfoo is the declarative one — a config file, a plugin library, a polished report, built to live in CI. Same target, same scenarios; only the framework changes. That’s the fair-test rule.

What each one actually broke

Here’s the uncomfortable headline: on this agent, only PyRIT landed a real hit. Its multi-turn escalation talked the agent into misusing its tool — the canary fired after three turns — with zero false positives. That’s the attack a single-turn scanner structurally cannot find, because it only exists across a conversation.

Garak reported three “hits” — and every one was a scoring artifact, not a real break. Its mitigation.MitigationBypass detector counted the agent’s benign, generic replies as jailbreaks, while its own dan.DAN detector passed all 127 attempts on the same responses. When one detector says “fully broken” and another says “fully resisted” on the identical output, that’s a false positive, and I flagged it as one. Garak’s single-turn battery never exercised the tool-misuse path at all.

Promptfoo went 0 for 40: every generated attack was resisted. The catch is why — every one of its iterative jailbreak strategies (composite, meta) is gated behind Promptfoo’s cloud generation service and simply refuses to run offline. So the strategies most likely to find the multi-turn break were the ones I couldn’t run locally. Clean report, but a real blind spot to know about.

Sidebar — the attacker that wouldn’t attack. PyRIT’s multi-turn attacks need a second LLM to invent the jailbreaks. The obvious move is to plug in the best model you have — so I plugged in a frontier model via its OAuth CLI. It refused: “I can’t help craft questions meant to make an assistant abandon safety rules.” The exact alignment that makes a model safe to ship makes it useless as a red-team engine. If you stand up PyRIT, budget for a red-team-permitted attacker model — don’t assume your production frontier key will do the job.

Noise and cost

A framework that flags every refusal as a jailbreak trains you to ignore it, and none of these attacks are free. Garak was the most thorough and the longest-running — 15 probes, 3,632 attempts — and it’s the one that produced false positives here. PyRIT was the most precise (~5 minutes, 0 false alarms) but the most engineering to drive. Promptfoo was the fastest route to a useful report (~6 minutes) but it leans on a judge model to grade success — you’re trusting one model’s opinion of whether another failed. Cheap and fast, worth a skeptical eye.

The scorecard and the verdict

Scored on the same seven-criterion rubric as every episode — install (15%), detection/efficacy (30%), signal quality (15%), performance (10%), usability (10%), docs (10%), value (10%):

  • PyRIT — 7.7. The only framework that modeled a patient, multi-turn adversary and actually found the tool-misuse break — also the most work to stand up. Reach for it when you’re serious about agent-specific attacks and have the engineers to compose it.
  • Promptfoo — 7.6. Easiest install, cleanest report, best docs, most CI-friendly — held back only because its strongest iterative strategies are locked behind the cloud service. The one to make part of a team’s daily workflow.
  • Garak — 6.9. A fast, broad, scanner-style first read — great for “where is this agent weak?” Just remember a wide net and a deep one are different instruments; read its detector hits critically.

For a real agent you’d layer them: Promptfoo or Garak in CI to catch regressions cheaply on every change, and a periodic deep PyRIT red-team before anything ships — because the multi-turn, tool-misuse attacks that actually matter are the ones a fast scan will never surface.
